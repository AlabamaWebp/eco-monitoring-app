# CSV Format

## Обязательные требования

- CSV должен содержать колонку `Дата`.
- Остальные колонки воспринимаются как датчики.
- Разделитель: запятая.
- Поддерживаемые кодировки: `UTF-8`, `UTF-8 with BOM`, `CP1251`.

## Поддерживаемые названия датчиков

- `CO2` / `co2` → `co2`
- `Влажность` / `humidity` → `humidity`
- `Температура` / `temperature` → `temperature`
- `Давление` / `pressure` → `pressure`
- `Освещённость` / `light` → `light`
- `Уровень шума` / `noise` → `noise`

## Пример CSV

```csv
Дата,CO2,Влажность,Температура,Давление,Освещённость,Уровень шума
2025-01-01 00:00:00,420,61,22.4,1012,450,38
2025-01-01 01:00:00,427,62,22.1,1011,430,39
```

## Как широкая строка превращается в длинный формат

Широкая строка:

```text
Дата=2025-01-01 00:00:00, CO2=420, Влажность=61, Температура=22.4
```

Превращается в несколько строк `measurements`:

```text
(2025-01-01 00:00:00, polygon_id=1, sensor_type=co2, value=420)
(2025-01-01 00:00:00, polygon_id=1, sensor_type=humidity, value=61)
(2025-01-01 00:00:00, polygon_id=1, sensor_type=temperature, value=22.4)
...
```

## Обработка ошибок и пропусков

- Если колонки `Дата` нет: импорт завершается ошибкой.
- Если есть неизвестные колонки датчиков: импорт завершается ошибкой.
- Пустые значения датчиков: пропускаются.
- Некорректные числовые значения: пропускаются.
- Все пропуски суммируются в `skipped_values`.
